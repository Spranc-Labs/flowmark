Iframe Content with PostMessage

This content is rendered inside an iframe. When you highlight text here, the PostMessageAdapter sends the highlight data to the parent window for storage.

How PostMessage Works

The PostMessage API enables secure cross-origin communication between windows. In this example, the iframe (child) sends highlight operations to the parent window, which stores them and responds back.

Save Highlight: When you create a highlight, the adapter sends a save_highlight message with the highlight data. The parent saves it and responds with confirmation.

Load Highlights: When the iframe loads, it sends a load_highlights message. The parent responds with all stored highlights, which are then rendered on the page.

Delete Highlight: Clicking a highlight sends a remove_highlight message with the highlight ID. The parent removes it from storage.

Use Cases

This pattern is perfect for browser extensions that inject iframes into web pages, or for applications that display user content in sandboxed iframes while maintaining centralized highlight storage.

Try it: Select any text above to create a highlight. Then use the reload button in the parent window to see your highlights persist!
